§01 manifesto

Every health system in 2026 is hand-rolling the same five things: drift monitors, audit trails, override tracking, consent ledgers, prospective evaluation harnesses. None of them should be bespoke. They are infrastructure — and infrastructure that is bespoke in a thousand hospitals is also broken in a thousand hospitals.

Asclepius is the missing layer underneath. It is not a model, not an EHR, not a governance committee. It is the substrate that turns any clinical-AI inference — a GPT-class scribe, a radiology CNN, an internal LLM agent, a rule engine — into something auditable, rate-limited, schema-checked, consent-aware, and replayable, through one runtime, with byte-equal artifacts a regulator can reverify offline.

§03 what asclepius is

Five primitives. One library. One library binding into every clinical AI you ship.

01

Signed Merkle ledger

Every inference, override, ground-truth label, and consent event becomes one row in an append-only log: BLAKE2b-256 payload hash, Ed25519 signature, prev_hash chained to the previous row. asclepius ledger verify walks the chain in O(n).

Ledger::append("inference.committed", actor, body)
02

Composable policy chain

Stateless guards run left-to-right on every inference: PHI scrubber, JSON-schema validator, length limit, clinical-action allowlist. The first BLOCK short-circuits with a structured rationale. Hosts write their own and push them in.

rt.policies().push(make_phi_scrubber());
03

Drift on the ground floor

Per-feature reference + current histograms. PSI, Kolmogorov–Smirnov, Earth Mover's Distance computed on demand against a fixed baseline so slow drift accumulates. Severity classifier exposed for calibration.

dm.observe("confidence", 0.74);
04

Consent & purpose-of-use

Eight codified purposes. Tokens granting purposes for a TTL. Every inference is scope-checked before any policy runs or any model is called. A missing or expired consent never reaches the wrapped tool.

consent.permits(patient, Purpose::triage)
05

Evidence bundles

USTAR tar containing manifest, ledger excerpt, per-model metrics, drift snapshot, override + ground-truth streams, public key, and an Ed25519 signature over the manifest. Verifiable offline with the public key alone.

harness.export_bundle(window, "evidence.tar")

§05 quickstart

Fifteen lines wraps any model in audit-grade safety.

// wrap any clinical model in 15 lines.
#include <asclepius/asclepius.hpp>
using namespace asclepius;

int main() {
  auto rt = Runtime::open("./asclepius.db").value();

  rt.policies().push(make_phi_scrubber());
  rt.policies().push(make_schema_validator(R"({"required":["soap"]})"));
  rt.policies().push(make_length_limit(10000, 4000));

  auto inf = rt.begin_inference({
    .model     = ModelId{"scribe", "v3"},
    .actor     = ActorId::clinician("dr.smith"),
    .patient   = PatientId::pseudonymous("p_9f1a"),
    .encounter = EncounterId::make(),
    .purpose   = Purpose::ambient_documentation,
  }).value();

  auto out = inf.run(transcript, my_scribe).value();
  inf.commit();
}

// wrap any clinical model from Go via the asclepius-svc sidecar.
package main

import (
    "context"
    "google.golang.org/grpc"
    pb "asclepius.health/proto/asclepius/v1"
)

func main() {
    conn, _ := grpc.Dial("asclepius-svc:7443", grpc.WithTransportCredentials(creds))
    cli := pb.NewAsclepiusClient(conn)
    ctx := context.Background()

    inf, _ := cli.BeginInference(ctx, &pb.BeginInferenceRequest{
        Spec: &pb.InferenceSpec{
            Model:     &pb.Id{Kind: pb.Id_MODEL,     Value: "scribe@v3"},
            Actor:     &pb.Id{Kind: pb.Id_ACTOR,     Value: "clinician:dr.smith"},
            Patient:   &pb.Id{Kind: pb.Id_PATIENT,   Value: "pat:p_9f1a"},
            Encounter: &pb.Id{Kind: pb.Id_ENCOUNTER, Value: "enc:01HX..."},
            Purpose:   pb.Purpose_AMBIENT_DOCUMENTATION,
        },
    })

    out, _ := cli.Run(ctx, &pb.RunRequest{
        InferenceId: inf.InferenceId,
        Input:       []byte(transcript),
    })
    cli.Commit(ctx, &pb.CommitRequest{InferenceId: inf.InferenceId})
    _ = out.Output
}
// wrap any clinical model from Java via the asclepius-svc sidecar.
import health.asclepius.proto.v1.*;
import io.grpc.ManagedChannelBuilder;

public class Wrap {
  public static void main(String[] args) throws Exception {
    var ch  = ManagedChannelBuilder.forAddress("asclepius-svc", 7443).useTransportSecurity().build();
    var cli = AsclepiusGrpc.newBlockingStub(ch);

    var spec = InferenceSpec.newBuilder()
        .setModel    (Id.newBuilder().setKind(Id.Kind.MODEL    ).setValue("scribe@v3"))
        .setActor    (Id.newBuilder().setKind(Id.Kind.ACTOR    ).setValue("clinician:dr.smith"))
        .setPatient  (Id.newBuilder().setKind(Id.Kind.PATIENT  ).setValue("pat:p_9f1a"))
        .setEncounter(Id.newBuilder().setKind(Id.Kind.ENCOUNTER).setValue("enc:01HX..."))
        .setPurpose  (Purpose.AMBIENT_DOCUMENTATION)
        .build();

    var inf = cli.beginInference(BeginInferenceRequest.newBuilder().setSpec(spec).build());
    var out = cli.run(RunRequest.newBuilder()
        .setInferenceId(inf.getInferenceId())
        .setInput(com.google.protobuf.ByteString.copyFromUtf8(transcript))
        .build());
    cli.commit(CommitRequest.newBuilder().setInferenceId(inf.getInferenceId()).build());
  }
}

§06 the verification ritual

A regulator should be able to verify your bundle on a laptop, offline, in under a second.

That is the bar Asclepius holds itself to. Every evidence bundle is a USTAR tar containing only canonical JSON, raw bytes, and an Ed25519 signature over the manifest. No network. No service. No vendor.

"If your audit story requires that we trust your servers, we don't trust your audit." — internal Asclepius design tenet

§07 runtime overhead, in microseconds

A trust substrate that costs more than the model it wraps would not be a substrate. Numbers we will defend.

Measured on a single core of an Apple M3 Pro, libsodium 1.0.20, SQLite 3.46 (WAL), nlohmann::json 3.11. Each row is the median over 1e5 iterations, with p99 in parentheses. The test harness lives in tests/bench/ and is part of CI.

policy chain 2.4 µs p99 · 7.1 µs

PHI scrubber + JSON-Schema validator + length limit, on a 600-byte SOAP transcript. Regex compilation is amortised via thread_local.

ledger append 28 µs p99 · 41 µs

Canonical JSON serialise, BLAKE2b payload + entry hash, Ed25519 sign, single SQLite INSERT. Hot mutex contention measured at < 1 % even at 10k QPS.

end-to-end run + commit 38 µs p99 · 62 µs

Excludes the wrapped model call. Microseconds, before the model returns. The wrapped model is ~four orders of magnitude slower; Asclepius is in the noise.

BLAKE2b throughput 1.2 GB/s single core · libsodium

Hashing the body of a 4 KB inference is ~3.3 µs, dominated by setup not throughput. Multi-core scales linearly; on an 8-core box, ~9 GB/s.

verify() a 1M-row chain 3.1 s cold · single thread

Walks every row, recomputes hashes, verifies every signature. A regulator can verify a year of a single-clinic deployment between two coffee sips.

evidence bundle export 82 ms 30-day window · 50k rows

Includes ledger excerpt, metrics, drift snapshot, override + ground-truth streams, manifest signing, USTAR tar emission. The output is a single, portable, signed file.

methodology Numbers are reproducible. cmake --preset=bench && ctest -V -L bench regenerates this table from your own machine. Reported deltas are eyeballed from the M3 Pro baseline; PRs that move a number by > 5 % must explain why.

§09 frequently asked, honestly answered

The questions a CTO asks before putting it in front of the AI Governance Office.

Is this just MLflow with extra steps?

No. MLflow tracks experiments — the offline lifecycle of training a model. Asclepius tracks production decisions — every inference a clinician saw, with the consent it ran under, the policies that gated it, the override that revised it, the ground truth that arrived two weeks later. The output of MLflow is a model registry. The output of Asclepius is a signed tar a regulator can verify offline. They live in the same week of your roadmap and solve different halves of the problem.

Why C++ and not Python?

Three reasons. Latency: the policy chain runs on every inference; we need microseconds before the model returns, not milliseconds. Determinism: replay must reproduce byte-equal artifacts across machines and years, which means canonical JSON serialization with no Python pickle drift. ABI stability: hospital procurement runs on five-year cycles. Python bindings ship in bindings/python; the core stays C++.

Does Asclepius see PHI? What about HIPAA?

By default the ledger stores hashes of inputs and outputs, not the content itself. PHI never enters the audit log unless an operator explicitly opts in via the encrypted body-store. Patient identifiers are pseudonymous tokens — Asclepius does not hold the MRN-to-token mapping; the EHR keeps that. The runtime is designed to be deployed inside the customer's HIPAA boundary; nothing calls home.

Will it work with my model? My EHR? My existing audit pipeline?

Yes. The wrapped model is a callable: (input) → output. Whether that's a remote GPT-class API, a CNN behind Triton, an internal LLM agent, or a rule engine — same wrapper. Identifiers (ActorId, PatientId, EncounterId) carry opaque strings, so they accept FHIR references, MRNs, OAuth subject claims, anything your stack already produces. The ledger emits canonical JSON; export it to your SIEM, your warehouse, or your bundle archive — your call.

What's the runtime overhead in production?

Microseconds for the policy chain (regex + JSON-Schema), tens of microseconds for the ledger append (one signed SQLite row). Sub-millisecond p99 for typical inferences. The dominant cost on the wrapped tool is still the model call itself, by orders of magnitude. Hashing throughput is ~1 GB/s per core via libsodium's BLAKE2b.

How do we deploy it — sidecar, in-process, or both?

Both, and they produce identical ledger entries. C++ and Python hosts link libasclepius directly. Java, Go, .NET, internal-language hosts talk to the asclepius-svc gRPC sidecar (schema in proto/v1/sidecar.proto). The SDK is the contract; the sidecar is a transport. A bundle exported from a sidecar deployment verifies against the same public key as one exported from an in-process deployment.

What does an "evidence bundle" actually mean to a regulator?

A USTAR tar. Inside: a JSON manifest listing per-file hashes, the relevant ledger excerpt, per-model metrics, drift snapshot, override and ground-truth streams, the public key, and an Ed25519 signature over the manifest. A regulator verifies it on a laptop, offline, using openssl pkeyutl -verify and sha256sum. No network, no portal, no "trust our servers." If the bundle doesn't verify with the public key your hospital published, the regulator stops reading.

Who's behind this? Is there a company?

The runtime is governed as an open-source project under Apache 2.0, with a standard PR-and-RFC contribution process. A managed-evidence service (multi-tenant bundle archival, regulator-portal handoff, HSM-backed signer) is a planned commercial offering — but the substrate is free, vendor-neutral, and runs entirely inside your trust boundary. The whole point is that the trust layer cannot be a SaaS lock-in.

How do I rotate the signing key?

Export an evidence bundle with the current key (this freezes the chain head into a checkpointed artifact), generate a new key, and continue. The bundle pins the ledger_head, so a verifier can reconstruct the world up to the rotation point with the old public key and after with the new one. Both keys live in your trust boundary; neither is ever held by Asclepius infrastructure.
