What this is. The policy chain in the C++ runtime composes input policies (PHI scrubber, length limit) and output policies (JSON-schema validator, action allowlist). Each policy returns one of allow, modify, or block. Whatever you compose here would be the JS-side equivalent of a single C++ IPolicy — small, stateless, and composable.

Three policy kinds.

  • Regex scrubber — every match becomes [REDACTED]; modify if any matches found, allow otherwise.
  • JSON-Schema validator — checks the encounter's output doc; allow if it conforms, block otherwise.
  • Custom function — a body that returns {decision, payload?, rationale?}. The expression has access to input (string) and output (parsed JSON).
