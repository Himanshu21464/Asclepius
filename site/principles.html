design tenets · seven · the why

What we will not trade away.

/governance describes how decisions get made. This page describes why they go a particular way. Each tenet below names a trade-off we accept. We make decisions by asking which tenet a change strengthens and which one it would weaken.

  1. Cryptographic provenance, not "trust us."

    Every byte we ask a hospital to act on must be verifiable offline by a third party. If the audit story requires that we trust your servers, we don't trust your audit. This is the substrate's reason to exist; if a feature would require shipping a trust assertion the customer cannot independently re-derive from bytes plus a public key, we don't ship it.

    trade-off We give up the convenience of opaque server-side aggregations. A regulator re-running our verifier on a laptop is the SLA we hold to.

  2. Vendor-neutral, by structure.

    The runtime accepts any callable as the wrapped model: an Epic plug-in, an in-house CNN, a remote GPT-class API, a rule engine. Identifiers are opaque strings; the runtime never holds the patient/MRN mapping; the audit format is one canonical-JSON shape regardless of who's on either side. Lock-in is not a feature.

    trade-off We give up the fast path of "deeply integrated with one EHR." We accept that our tightest customer integrations will be slower to ship.

  3. Audit-grade or it does not ship.

    Every public method either appends to the ledger, contributes to a verifiable metric, or is intentionally outside the audit boundary (and documented as such). A feature that produces a clinical decision but no audit trail is a worse-than-nothing feature: it lets the appearance of safety substitute for the real thing. We refuse to ship one.

    trade-off Some convenient features stay unbuilt because we cannot make them auditable without ugly compromises. We accept that the runtime is smaller as a result.

  4. One canonical encoding, forever.

    Canonical JSON for ledger bodies. BLAKE2b-256 for hashes. Ed25519 for signatures. USTAR for evidence bundles. We pick once, document the choice, and version-bump only when we have a stronger primitive widely available. Replay must reproduce byte-equal artifacts across machines and across years.

    trade-off We give up the option of "let's switch to BLAKE3 because it's faster" mid-line. Performance gains we can capture without changing the format are welcome; format churn never is.

  5. Substrate, not SaaS.

    The runtime runs entirely inside the customer's trust boundary. There is no call home, no anonymous telemetry, no opt-out behavior — there is no opt-in either, because the code does not contain network egress for analytics. The site that markets it holds itself to the same posture: /cookies, DNT-policy, and the /colophon document the discipline.

    trade-off We give up the visibility into customer deployments that a SaaS gives. We accept that we will learn about issues later, from explicit reports rather than implicit metrics.

  6. Specific over clever.

    We pick boring, specific tools where one will do: SQLite WAL for the ledger backend, libsodium for the primitives, doctest for testing, vanilla JS for the site. Every layer of cleverness is a layer of attack surface and a layer that new contributors must understand before they can ship a fix. Three similar lines beats one premature abstraction.

    trade-off We give up the elegance of frameworks. We accept that the codebase has more small, repetitive places than it would have under a clever DSL.

  7. Patients, not dashboards.

    Everything Asclepius does eventually runs next to a clinician, eventually runs next to a patient. We design defaults for the patient, not the dashboard. If a feature would make a governance committee happier but a patient worse off, we don't ship it. The clinical-action filter denies by default. Consent is required by default. PHI is hashed by default. Bundles are signed by default. Convenience is opt-in.

    trade-off We give up the fast onboarding curve of "opt in to safety later." We accept slower onboarding for stronger defaults.

These principles fit on one page on purpose. A principles document longer than its codebase principles nothing. When a tenet starts to lie about how we actually behave, we update the tenet — not the behavior. The git log of this page is the canonical history; if a maintainer cannot point to a recent commit demonstrating one of the principles above, that's a flag for the next contributor sync.
