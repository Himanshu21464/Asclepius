Everything we did not write.

Asclepius itself is Apache 2.0. The runtime links a small, deliberately curated set of dependencies; the website embeds three font families. Every one is listed below with its license, version, and the reason we picked it. If anything is missing, that's a bug — email hello@asclepius.health.

Runtime dependencies

Linked into libasclepius at build time. Each is declared in cmake/Dependencies.cmake and pinned by tag.

libsodium ISC 1.0.20BLAKE2b-256 hashing and Ed25519 signatures. Chosen for its no-config, constant-time discipline and wide platform support.
SQLite Public domain 3.46The append-only WAL backend for the Merkle ledger. Single-file, zero-config, regulator-friendly.
fmt MIT 10.2Type-safe formatting. Replaces printf in error paths and the CLI.
spdlog MIT 1.13Structured logging — used sparingly; the ledger is the log of record.
nlohmann/json MIT 3.11Canonical-JSON serialization for ledger entries. The hash function depends on key-sorted output; nlohmann's dump guarantees that.
doctest MIT 2.4Test framework. Single header, fast compile, drives all 312 unit cases.
CLI11 BSD 3-Clause 2.4The argument parser behind the asclepius CLI.

Binding dependencies

Optional — pulled only when the corresponding binding is enabled.

pybind11 BSD 3-Clause 2.13The Python-binding glue. Header-only, ABI-stable across Python 3.10–3.12.
cxx Apache 2.0 / MIT 1.0 (planned)Rust bindings via cxx. Not yet shipped.
protobuf + grpc BSD 3-Clause 5.29 + 1.66 (sidecar)Pulled only when ASCLEPIUS_BUILD_SVC=ON. The sidecar is the only place we touch them.

What this doesn't ship

No analytics SDKs. No A/B testing frameworks. No customer-data platforms. No third-party CDN-served JavaScript bundles. The only over-the-wire dependency at runtime is the Google Fonts stylesheet — and we honor a preconnect + preload hint so first paint is not blocked on it. If you are reading this on a privacy-respecting browser and you block Google Fonts, the page falls back to system-stack serif/sans/mono and reads cleanly.
