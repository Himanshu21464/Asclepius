Wrap an inference C++20

#include "asclepius/asclepius.hpp"
using namespace asclepius;

auto rt = Runtime::open("./asclepius.db").value();
rt.policies().push(make_phi_scrubber());
rt.policies().push(make_schema_validator(R"({"required":["soap"]})"));

auto inf = rt.begin_inference({
  .model     = ModelId{"scribe", "v3"},
  .actor     = ActorId::clinician("dr.smith"),
  .patient   = PatientId::pseudonymous("p_9f1a"),
  .encounter = EncounterId::make(),
  .purpose   = Purpose::ambient_documentation,
}).value();

auto out = inf.run(transcript, my_scribe).value();
inf.commit();

Wrap an inference Python

import asclepius as ax

rt = ax.Runtime("./asclepius.db")
rt.policies.push(ax.policies.phi_scrubber())
rt.policies.push(ax.policies.schema_validator({"required": ["soap"]}))

with rt.begin_inference(
    model     = ax.Model("scribe", "v3"),
    actor     = ax.Actor.clinician("dr.smith"),
    patient   = ax.Patient.pseudonymous("p_9f1a"),
    encounter = ax.Encounter.new(),
    purpose   = ax.Purpose.AMBIENT_DOCUMENTATION,
) as inf:
    soap = inf.run(transcript, my_scribe)
    inf.observe_drift("confidence", 0.74)

Verify a bundle without our binary

# 1. extract
tar xf evidence.tar

# 2. authenticate the manifest
{ printf '\x30\x2a\x30\x05\x06\x03\x2b\x65\x70\x03\x21\x00';
  cat public_key; } | openssl base64 \
  | sed -e '1i-----BEGIN PUBLIC KEY-----' \
        -e '$a-----END PUBLIC KEY-----' > pk.pem

openssl pkeyutl -verify -pubin -inkey pk.pem \
    -sigfile manifest.sig -in manifest.json
Signature Verified Successfully

# 3. confirm every file's hash
jq -r '.files | to_entries[] | "\(.value)  \(.key)"' \
    manifest.json | sha256sum -c

Built-in policies composable

  • make_phi_scrubber() — SSN, phone, email, MRN, dates → modify.
  • make_schema_validator(schema) — JSON-Schema subset on output → block on violation.
  • make_clinical_action_filter(allowlist) — rejects non-allowlisted action codes.
  • make_length_limit(in, out) — bounded payload size.

Verdicts compose left-to-right: allow → modify → block. The first block short-circuits with a structured rationale.

Ledger entry spec.md

{
  "seq": 1234,
  "ts": "2026-04-25T17:22:25Z",
  "actor": "clinician:dr.smith",
  "event_type": "inference.committed",
  "tenant": "kp-northwest",
  "prev_hash": "<32 bytes hex>",
  "payload_hash": "<32 bytes hex>",
  "body": { /* canonical-JSON event payload */ },
  "signature": "<64 bytes hex>",
  "key_id": "<16 hex>"
}

Hash: BLAKE2b-256(event_type || 0x1E || canonical_json(body)). Signature: Ed25519 over seq|ts|prev|payload|actor|event_type|tenant|body.

Key custody three options

  • file 0600 — default, fine for a private VM
  • cloud KMS — encrypted-at-rest, IAM-gated
  • HSM (PKCS#11) — key never leaves device, ~5–10× slower per signature, required for FedRAMP-High

Rotate by writing the new public key to the meta table; new entries sign with the new key_id; old entries continue to verify against the old key.

Threat model TL;DR

A1
internal log tampering → chain detects
A2
malicious model output → policy chain
A3
replay → seq monotonicity
A4
bundle forgery → manifest sig
A5
PHI side-channel → hashes only by default
A6
undetected drift → frozen baseline

Out of scope: signing key compromise, host root, insider with both DB write + key access, post-quantum, bundle exfiltration.

What not to expect

  • Asclepius is not an AI scribe.
  • Asclepius is not an MLOps platform.
  • Asclepius is not an EHR.
  • Asclepius is not a cloud service. Self-hostable, period.
  • Asclepius does not reach the model — you call it; we wrap it.
