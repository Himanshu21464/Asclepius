§01 · annotated · cited · standing on shoulders

Where the primitives come from.

Asclepius does not invent its own crypto, drift maths, or interop schemas — it composes proven ones. This bibliography is the canon the substrate is built on. Each entry is annotated with what we use it for and which ADR or section depends on it. Read the originals; we did.

§ 01 · cryptographic primitives

Crypto we chose to build on.

The substrate signs and hashes on the hot path of every clinical inference. Performance, correctness, and longevity all matter. Each of the primitives below has decades of cryptanalytic scrutiny — that's the point.

Merkle, R. C. (1979). Secrecy, authentication, and public key systems. PhD thesis, Stanford. (See also: A Certified Digital Signature, CRYPTO 1989.)

The Merkle tree — log integrity for forty-five years.

The original construction. Asclepius's ledger is a hash chain (a degenerate Merkle tree where each node has one child); evidence bundles include a full Merkle root over the entries. The mathematical guarantee — that any tampering leaves a verifiable trace — is unchanged since 1979.

Used in: architecture, spec.

Bernstein, D. J., Duif, N., Lange, T., Schwabe, P., Yang, B.-Y. (2012). High-speed high-security signatures. Journal of Cryptographic Engineering 2(2):77–89.

Ed25519 — the signature scheme we sign with.

Defines the curve, the encoding, the deterministic-nonce derivation. Asclepius uses libsodium's reference implementation. Deterministic signatures are an audit feature, not a side effect — same input + key always produces the same sig, which makes replay-comparison decisive.

Used in: ADR-001.

Aumasson, J.-P., Neves, S., Wilcox-O'Hearn, Z., Winnerlein, C. (2013). BLAKE2: simpler, smaller, fast as MD5. ACNS 2013.

BLAKE2 — the hash the ledger leans on.

BLAKE2b-256 in libsodium gives ~1 GB/s/core throughput with no length-extension footgun. Every prev_hash, input_hash, output_hash, and bundle manifest entry uses it.

Used in: ADR-002.

NIST. (2024). FIPS 204 — Module-Lattice-Based Digital Signature Standard (ML-DSA).

ML-DSA — the post-quantum migration target.

The PQ-safe successor to Ed25519. Bigger signatures (≈ 3 KB at security level 2), but Shor-resistant. Asclepius's wire format already prefixes signatures with a scheme identifier, so the migration is additive — see ADR-010.

Used in: ADR-010 (proposed).

Bernstein, D. J., Lange, T. (curators). libsodium / NaCl. Maintained by Frank Denis and contributors.

libsodium — the implementation we trust to be right.

Misuse-resistant API, audited by multiple independent reviews, available everywhere the substrate ships. We do not roll our own. Every crypto_sign_* and crypto_generichash call is libsodium's.

https://doc.libsodium.org/

§ 02 · provenance & transparency logs

Prior art for auditable systems.

Asclepius is not the first append-only signed log. The conceptual framework — verifiable, replayable, externally-auditable — is shared with Certificate Transparency, in-toto, and the SLSA supply-chain framework. We borrow shamelessly.

Laurie, B., Langley, A., Käsper, E. (2013). RFC 6962 — Certificate Transparency.

CT — public, append-only, gossiped.

The specification that proved you can run an audit log at internet scale and have third parties verify it without trusting the operator. Asclepius's "publish the Merkle root in the public changelog feed" pattern is borrowed directly.

Trillian project, Google. (2015–). Trillian: general-purpose append-only log.

Trillian — the operational reference.

An open-source implementation of a CT-style log with consistency proofs, gossip, and operations tooling. The verify_chain implementation in Asclepius mirrors Trillian's range proofs.

https://github.com/google/trillian

Torres-Arias, S., Ammula, A. K., Curtmola, R., Cappos, J. (2019). in-toto: providing farm-to-table guarantees for bits and bytes. USENIX Security.

in-toto — supply-chain attestation that composes.

The conceptual ancestor of "every step in a process is signed by the actor who performed it, in a way that downstream consumers can verify end-to-end." That is exactly the chain Asclepius produces for clinical inferences.

Open Source Security Foundation (OpenSSF). (2022–). SLSA — Supply-chain Levels for Software Artifacts.

SLSA — the L1/L2/L3 ladder we mirror.

The graduated-conformance model in Asclepius's conformance suite (L1 audit · L2 drift · L3 prospective evidence) is structurally similar to SLSA's level-up framework. Conformance is a ladder, not a checkbox.

https://slsa.dev/

§ 03 · distribution drift

How the drift monitor measures.

The substrate watches three statistics per feature: PSI for binned distributions, KS for continuous, EMD for ordinal. Each comes from an established literature. The substrate's job is to compute them honestly and bind their values to the ledger; the threshold-setting and remediation belong to the operator.

Wu, N., Zhao, J. (2007). Population Stability Index methodology. Discover Card credit-risk technical note (widely cited industrial reference).

PSI — the binned-distribution shift index.

The metric of choice for monitoring credit-risk model drift, adopted broadly across industries. Cheap to compute, interpretable: PSI < 0.1 stable, > 0.25 distribution change, > 0.5 major shift. Asclepius's default threshold is 0.25.

Used in: PB-04.

Kolmogorov, A. N. (1933). Sulla determinazione empirica di una legge di distribuzione. Giornale dell'Istituto Italiano degli Attuari 4:83–91. — Smirnov, N. (1948).

KS test — continuous-feature shift.

For features that are continuous (vitals, lab values), PSI's bucketing introduces noise. The Kolmogorov–Smirnov statistic — the supremum of |F1(x) − F2(x)| — is binning-free and well-understood.

Rubner, Y., Tomasi, C., Guibas, L. J. (2000). The Earth Mover's Distance as a Metric for Image Retrieval. Int. J. Computer Vision 40(2):99–121.

EMD — ordinal drift, where order matters.

For ordinal features (severity scores, age bands), shifts that move probability mass adjacent vs distant rungs matter very differently. EMD captures that by penalising the work required to transport one distribution to the other.

Niculescu-Mizil, A., Caruana, R. (2005). Predicting good probabilities with supervised learning. ICML.

Calibration — the other drift.

Distribution-shift can be silent; calibration loss is the louder cousin. The Evaluation Harness writes reliability curves and ECE to every QA bundle, which is what regulators look at first.

§ 04 · clinical-AI accountability

The literature the substrate enacts.

Many of the practices Asclepius automates were proposed in writing years before the tooling caught up. Model cards, datasheets, and audit frameworks defined the artefacts; the substrate just makes them cryptographically real.

Mitchell, M. et al. (2019). Model Cards for Model Reporting. FAccT 2019.

Model cards — declared once, bound to every inference.

The model-card framework standardised "what the model is, where it works, where it fails." Asclepius binds the model's id + weights_sha256 to every entry, which makes the corresponding model card resolvable from any row.

Gebru, T. et al. (2018). Datasheets for Datasets. CACM 64(12).

Datasheets — the training-data contract.

Out-of-substrate, but motivates the discipline that the substrate enforces: commit to what the model has seen, then prove what it does on what it hasn't.

Sendak, M. P., Gao, M., Brajer, N., Balu, S. (2020). Presenting machine learning model information to clinical end users with model facts labels. NPJ Digital Medicine 3(1).

Model Facts — at the bedside.

Demonstrates that simple, nutrition-label-style summaries of model performance are usable at the point of care. The substrate is what makes those labels reliably truthful — the underlying numbers are pulled from the chain, not asserted.

Wiens, J. et al. (2019). Do no harm: a roadmap for responsible machine learning for health care. Nature Medicine 25(9):1337–1340.

Do no harm — the operational imperative.

The framework that crystallised "deployed clinical-AI is a regulated medical activity, not a research output." Every primitive in Asclepius is downstream of that imperative.

Raji, I. D., Smart, A., White, R. N., Mitchell, M., Gebru, T., et al. (2020). Closing the AI accountability gap: defining an end-to-end framework for internal algorithmic auditing. FAccT.

The accountability gap we are filling.

Defines the auditing artefacts an organisation must produce to hold itself accountable for an AI system. The substrate's evidence-bundle export is, deliberately, a turn-key version of those artefacts.

§ 05 · interop standards

The specs we speak.

Substrate code only matters if it can plug into the world. These are the interop standards Asclepius implements verbatim, so the operator does not have to glue our shape to theirs.

HL7 International. (2023). FHIR R5 — Fast Healthcare Interoperability Resources.

FHIR R5 — the EHR contract.

The substrate's interop layer (asc::interop::FhirBundle) parses FHIR Bundles, derives runtime context, and emits DocumentReferences whose identifier.value is the ledger seq.

Used in: FHIR sidecar recipe.

NEMA. (2024). DICOM PS3.x — Digital Imaging and Communications in Medicine.

DICOM — the imaging contract.

DICOM files carry both pixels and PHI tags. The DicomStrip policy strips identifying tags (PatientName, PatientID, AccessionNumber, etc.) before the bytes reach the model.

IHE Health IT Profiles. (ongoing). ITI/PCC technical frameworks.

IHE — the workflow profiles.

Where FHIR specifies the resource shapes, IHE specifies how they flow between actors. Asclepius's actor scheme (clin:, svc:, sys:, agent:) is informed by IHE actor terminology.

IEEE / ISO. (1988). POSIX.1 — IEEE Std 1003.1, Annex on USTAR archive interchange.

USTAR tar — portable forever.

The lowest-common-denominator archive format that opens with a one-decade-from-now tool. Evidence bundles use it; recipients verify with stock tar and jq.

Used in: ADR-006.

JSON Schema Specification. (2020). Draft 2020-12.

JSON Schema — the entry shape contract.

The audit-entry schema is a Draft 2020-12 document. Validators in any language can verify that any candidate entry conforms before signing. The schema is an artefact, not just documentation.

Used in: /schemas/audit-entry.v1.json.

What we are not citing.

We are not citing Bitcoin. We are not citing "blockchain for healthcare" papers. The substrate uses a hash chain and digital signatures because Merkle described them in 1979 and Bernstein-et-al. described how to do them efficiently in 2012; cryptography existed before tokens, and will exist after them.

If a paper that ought to be on this list is missing, please open an issue. We would much rather extend the canon than gatekeep it.
