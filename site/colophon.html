colophon · about this site

The site, by the same posture as the substrate.

Hand-rolled. No framework. No bundler. No analytics. Every page validates against a single Python script. Every byte of every response is published in a SHA-256 manifest at /asset-manifest.json and the footer pill on every page verifies the page you are reading right now.

type specimen

The Linux kernel for clinical AI.

Display in Instrument Serif, italics doing the rhetorical work. Long-form prose in Newsreader, four weights only — 300, 400, 500, 600. Code, hashes, and meta in JetBrains Mono, deliberately neither pretty nor ugly.

Type stack

display Instrument Serif, italic where structural — never roman-set the wordmark. Loaded from Google Fonts; display=swap so first paint never blocks. SIL OFL 1.1.
prose Newsreader 300/400/500/600, both upright and italic. Comfortable at 17px, still readable at 14px. SIL OFL 1.1.
mono JetBrains Mono 400/500/600. Used for hashes, identifiers, code samples, telemetry strips, kbd chips. SIL OFL 1.1.
fallback If a visitor blocks Google Fonts the page falls through to system serif/sans/mono. We tested with fonts.googleapis.com blocked: layout holds, italics still italicize.

The light-theme palette inverts surface and ink and uses the canonical-deep variants of primary and accent. No pure-white backgrounds, no pure-black text — both readings are tinted toward primary.

Build pipeline

Three Python helpers, one Makefile, zero JavaScript build step. Default flow on every edit: 

$ make all
asclepius site validator — 32 html files
✓ all clean
wrote asset-manifest.json — 61 files
wrote search-index.json — 31 pages
✓ site refreshed

check_html.py walks every page for broken links, duplicate IDs, missing alts, heading-hierarchy violations, and metadata gaps; regen_manifest.py recomputes SHA-256 of every served file; regen_search.py rebuilds the page index. build_docs.py renders the markdown docs into themed HTML pages. Make targets coordinate.

JavaScript discipline

Nine modules, ~2.4k lines, no transpilation, no bundling, no source maps. Each file is self-contained and IIFE-wrapped:

Self-attestation

The substrate's thesis is that audit must be cryptographically verifiable, not "trust us." We hold the site to the same bar. Every file the server returns has its SHA-256 published in /asset-manifest.json. The pill at the bottom-right corner of every page — the small ◇ — recomputes the hash of the <main> you are reading and shows you whether it matches the manifest entry. Click it to verify what we just served you.

This is the same posture as the runtime, applied to the site that markets it. There is no coherent way to ask hospitals to trust an audit substrate while shipping their executives a marketing site that runs Google Tag Manager.

What didn't make the cut

We tried a Tailwind variant: too much HTML noise. We tried a build with esbuild: the speed gain wasn't worth the cognitive cost of a build step. We tried an MDX-rendered docs path: the markdown source is the source of truth, and HTML rendered from it stayed simpler.

The site has remained at zero runtime dependencies (other than Google Fonts, which is opt-out via /cookies) for every iteration. Adding one is a code change, a PR, and an RFC.

Lessons

— Validating before publishing is faster than fixing in production. make all on every change.

— Italic accents do most of the rhetorical work; they survive theme inversion gracefully and cost nothing on render.

— A small, hand-curated command palette beats live full-text search for a site of 32 pages. At 100 pages we may revisit.

— Self-attestation is harder than runtime attestation, because pages get edited often and the manifest goes stale. regen_manifest.py in make all closes the loop.
