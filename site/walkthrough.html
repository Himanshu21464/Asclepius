§02 the substrate appears

A thin layer between.

Drop one C++ binary between every clinical-AI tool and the EHR it speaks to. The tool keeps its model. The EHR keeps its workflow. Asclepius keeps the record.

§03 wrap an inference

Four filters, one trace.

Each call passes the same chain: scrub PHI, validate schema, enforce length, observe drift. Composable, reorderable, tracked per row.

§04 the signed ledger

Five rows, one chain.

Each row is BLAKE2b-256 of the canonical body, Ed25519-signed, and prev_hash-chained to the row before it. Walk it offline; flip a byte and the chain breaks.

§08 what you actually deploy

Three files. Sixty seconds.

No SaaS. No call-home. No new platform. Drop the binary on a node next to the AI, hand it the SQLite path and the keypair, and start writing rows.

binary

asclepius

One C++20 ELF. Statically linked libsodium + sqlite. ≈ 12k SLOC of substrate.

state

ledger.db

Single SQLite file, WAL mode. Append-only ledger, indexed by seq + actor + tenant.

identity

signer.key

Ed25519 keypair on disk, or HSM if you have one. Public half ships in every bundle.
