Read someone else's ledger.

The Asclepius playground builds a chain in front of you. The Explorer receives one. Drop a ledger_excerpt.jsonl from a real bundle, or paste any JSONL ledger snippet, and walk it line by line — chain, payload-hash recompute, signature stand-in. Everything happens in your browser; nothing is uploaded.

What this checks. The Explorer recomputes SHA-256(event_type || \x1e || canonical_json(body)) for each line and compares it to the entry's declared payload_hash. It also walks prev_hash through the file: each entry's prev_hash must equal the previous entry's entry_hash. Either mismatch lights the row red.

What it does not check. The signature. Verifying signatures requires the operator's public key, which lives outside the JSONL — see verify-bundle for the offline ritual that does include it.

Hash function caveat. The C++ runtime uses BLAKE2b-256. Web Crypto doesn't ship BLAKE2b, so this in-browser tool checks SHA-256 instead — pre-process the file with asclepius ledger inspect --rehash sha256 … if you want byte-equal verdicts.
