Microseconds, not milliseconds.

Real numbers from the C++20 runtime, on a single core of a commodity workstation (Ubuntu 25.x, GCC 15, libsodium 1.0.18, SQLite 3.46 in WAL mode). The wrapped model still dominates the request budget — Asclepius adds a microseconds- thick layer on top of it, signed and committed.

Cryptographic primitives are not the bottleneck.

BLAKE2b-256 hashing runs at L1-friendly speeds. Ed25519 signing pays roughly 16 µs per inference and verification 45 µs — both well below the latency of any clinical-AI model worth wrapping. None of these numbers depend on a network round-trip; everything happens in the calling thread.

Runtime: ledger, policy, end-to-end.

The policy chain here composes the PHI scrubber + length limit. The ledger append is a single signed SQLite WAL insert with full Merkle bookkeeping. Numbers are single-thread; appending fans out cleanly across many readers.

Where the 87 µs goes.

Stack-graph of where one wrapped inference (p50) actually spends its time. Everything below the wrapped model call is "Asclepius overhead" — the cost of being audited.

policy in 25 µs · 29%
model ~2 µs · 2%
policy out 25 µs · 29%
sign + ledger 31 µs · 36%
misc 4 µs · 4%

A real clinical model is hundreds of milliseconds (CNN inference) to tens of seconds (LLM scribe). The wrap layer is round-off error on that budget — and it buys you a signed, replayable, regulator-verifiable audit trail.

Reproduce them yourself.

The bench harness is examples/04_bench/main.cpp. The full JSON output of the run that produced this page lives at assets/bench.json for direct inspection.

Run it

From the project root after a normal build:

cmake --build build --target example_bench
./build/examples/04_bench/example_bench --n 30000 \
    > my-bench.json

Methodology

30,000 iterations for hash / sign / verify / policy. 2,000 rows for ledger append. 1,000 wrapped inferences for end-to-end timing. Single thread. Median of three runs reported. No JIT — straight-line C++20 with -O2.

What is not measured

Network sidecar latency (gRPC), encrypted-body-store IO (planned), HSM-backed signing (slower per-op but parallel-friendly), Postgres-backed ledger (planned), and any model call.

Honest caveats

Numbers move with hardware. AVX-512 boxes hash faster, ARM64 signs slightly slower. The headline claim is the shape: microseconds, not milliseconds. We will publish a multi-arch matrix in v0.2.
