glossary · v0.1.0

Every term, defined once.

This is the canonical source. Other pages link here rather than redefine. Press / or use the search to filter; deep-link any term by appending its anchor (e.g. #blake2b-256).

/

A

ambient documentationpurpose
The clinical-AI category that records and structures a patient encounter automatically — voice-to-SOAP. One of the eight Purposes the runtime enforces consent against.
attestationprocess
The act of vouching for a byte sequence with a cryptographic signature. Every Asclepius ledger entry is an attestation by the operator's signing key over the entry's canonical encoding.
ABDMregime · India
Ayushman Bharat Digital Mission — India's national digital-health programme run by the National Health Authority. Provides the ABHA patient-id namespace, the HFR facility registry, and the HPR healthcare-professional registry. The India profile maps these into PatientId, TenantId, and ActorId respectively, without changing the substrate's invariants. See ADR-013.

B

BLAKE2b-256hash
A 256-bit cryptographic hash, optimised for software. libsodium's crypto_generichash with default parameters. Used for every payload and entry hash in the runtime; ~633 MB/s on a single core (see benchmarks).
BAAcontract
Business Associate Agreement. Required by HIPAA whenever a vendor handles PHI on a covered entity's behalf. Asclepius is intentionally deployable inside the customer's HIPAA boundary so the project itself does not need a BAA with each customer.
evidence bundleartifact
A USTAR .tar containing manifest + ledger excerpt + metrics + drift snapshot + override stream + ground-truth stream + the public key + an Ed25519 signature over the manifest. Verifiable offline with openssl + jq + sha256sum alone.

C

canonical JSONencoding
UTF-8, sorted keys, no insignificant whitespace. Two canonical encodings are byte-equal iff the underlying values are JSON-equal. Used by every payload hash so re-running the same encoding on different machines years apart yields the same bytes.
CDSSacronym
Clinical Decision Support System. A class of clinical software that surfaces evidence-based guidance to clinicians. Asclepius wraps CDSS the same way it wraps any other model — the policy chain doesn't care.
conformance suiteL1 / L2 / L3
The third-party-runnable test battery that validates a vendor integration. L1 = audit, L2 = drift, L3 = prospective-evidence. Each level strictly extends the previous.
ConsentArtefacttype · India
JSON-serialisable consent record on the wire between HIPs / HIUs and the ABDM Gateway. Bidirectionally maps to ConsentToken via artefact_from_token / token_from_artefact; serialised by to_abdm_json. See /india §02.
consent tokentype
An opaque identifier issued by ConsentRegistry::grant() that pairs a PatientId with one or more Purposes and a TTL. Required by begin_inference before any model is called.

D

DPDP § 7law · India
Digital Personal Data Protection Act, 2023, § 7 — the break-glass clause that allows clinical processing of personal data without prior consent in a medical emergency. Why EmergencyOverride exists; the kernel codifies the requirement, not the regulation.
driftconcept
The slow, often invisible shift in input distribution that causes a deployed model's accuracy to decay over time. Asclepius computes PSI, KS, and EMD against a frozen baseline so slow drift accumulates rather than averaging out.

E

Ed25519signature
An EdDSA signature scheme over Curve25519. 32-byte public key, 64-byte signature, deterministic, fast, side-channel resistant. The default Asclepius signer; chosen over ECDSA because RNG failures during signing don't leak the secret key.
EncounterIdtype
A strong-typed identifier for a clinical encounter. Wraps an opaque string body — accepts FHIR Encounter/<id> references, internal numeric ids, or a fresh ULID-shaped id from EncounterId::make().
EMDstatistic
Earth Mover's Distance — for 1D distributions, the cumulative absolute difference between two CDFs. Reported alongside PSI and KS by DriftMonitor::report() for triangulation.
EmergencyOverrideprimitive · India
The kernel primitive enforcing mandatory backfill within a configurable window after a break-glass activation. Default 72h aligns with DPDP § 7; set_backfill_window() moves it. is_overdue() surfaces tokens whose deadline has passed; see /india §04.

F

FamilyGraphprimitive · India
Multi-party consent graph encoding adult-for-elder, parent-for-minor, guardian, and spouse relationships. A token issued by a proxy is honoured iff a valid edge exists. Edge writes fire an observer so the audit ledger mirrors the graph; see /india §03.
FHIRstandard
Fast Healthcare Interoperability Resources. The HL7 RESTful health-data exchange standard. Asclepius is FHIR-aware in the narrow sense that it can extract Patient/Encounter/Practitioner references and Purpose-of-use codes from a Bundle — but it is not a FHIR server. Use HAPI/Aidbox for that.

G

ground truthsignal
The "what actually happened" label that arrives sometime after a model's prediction. Asclepius's EvaluationHarness::attach_ground_truth ingests these and joins them by inference id so the L3 conformance can compare.

H

hashfunction
A deterministic function from arbitrary bytes to a fixed-size digest. Asclepius uses BLAKE2b-256 specifically; the in-tab demo uses SHA-256 because Web Crypto doesn't ship BLAKE2b. See the crypto toolbox.
HIP / HIUrole · India
Health Information Provider (the source of records — a hospital, a lab) and Health Information User (the consumer — a clinician, a tele-consult platform). The two endpoints that exchange a ConsentArtefact via the ABDM Gateway.
HIPAAlaw
The US federal Health Insurance Portability and Accountability Act, including its Privacy Rule (1996) and Security Rule (2003). Sets the floor for how PHI is handled. Asclepius is designed to deploy inside the customer's HIPAA boundary; the project itself never touches PHI.
HSMhardware
Hardware Security Module. A tamper-resistant device that holds signing keys and performs operations without exposing them. Asclepius's KeyStore is an interface; an HSM-backed implementation (PKCS#11 reference adapter) is on the roadmap.

K

KSstatistic
Kolmogorov–Smirnov statistic. The maximum absolute difference between two empirical CDFs. Reported alongside PSI by Asclepius's drift monitor.

L

ledgercore artefact
The append-only, Merkle-chained, Ed25519-signed event log that Asclepius writes to. Persisted to SQLite WAL by default. asclepius ledger verify walks the chain in O(n).

M

Merkle chainstructure
A linked list where each node includes a hash of the previous node. Mutating any past entry breaks the chain that follows. Asclepius's ledger is a chain, not a tree — a tree would batch entries and need root-recompute semantics; an append-only log doesn't.
MRNidentifier
Medical Record Number. A patient's identifier inside one health system. Asclepius does not hold the MRN; the EHR maps MRN → pseudonymous PatientId on its side.

O

overrideevent
A clinician's structured rejection or revision of a model's output. Captured with rationale by Inference::capture_override. The override rate is a leading indicator of model fitness.

P

PatientIdtype
A strong-typed pseudonymous identifier. Constructors include PatientId::pseudonymous and PatientId::fhir. Asclepius never sees the patient's MRN; the EHR maps the MRN to a token before passing it to the runtime.
PHIregulated data
Protected Health Information. Any information about an identifiable patient's health that a covered entity holds. Asclepius's PHI scrubber redacts obvious patterns (SSN, MRN, phone, email, dates of birth) before the model sees them; production hosts compose this with a model-based scrubber for richer coverage.
PSIstatistic
Population Stability Index. Σ (p_i − q_i) · ln(p_i / q_i). Conventional severity bands: <0.10 stable, 0.10–0.25 minor, 0.25–0.50 moderate, ≥0.50 severe. Asclepius computes PSI against a frozen baseline so slow drift accumulates.
Purposeenum
One of fourteen codified clinical purposes — the original eight (ambient_documentation, diagnostic_suggestion, triage, medication_review, risk_stratification, quality_improvement, research, operations) plus six added in round 90 for the India profile (prescription_resolution, second_opinion, specialist_referral, billing_audit, longitudinal_outcomes_research, emergency_clinical_access). Adding a new one is a governance event, not a string change; see ADR-014.

R

Result<T, Error>type
Asclepius's success-or-error return type. No exceptions in the hot path; callers either get the value or a structured Error with a code from ErrorCode.

S

SOAP noteformat
Subjective, Objective, Assessment, Plan — the canonical structured clinical-encounter note. The Asclepius playground's fake scribe emits SOAP-shaped JSON; the schema validator policy gates that shape on output.
SQLite WALmode
Write-Ahead Log mode — single-writer, many-reader, durable, no daemon. Asclepius's only ledger backend as of v0.4.0; one DB file per sidecar, single-node by design.

T

tamperedstate
An entry whose body has been altered after it was signed. Detected by Ledger::verify(): the recomputed payload hash no longer matches the one signed in the entry header.

U

ULIDidentifier
Universally Unique Lexicographically Sortable Identifier. Encodes a 48-bit timestamp + 80 bits of randomness in 26 chars of Crockford base32. EncounterId::make() uses the ULID shape for time-orderable encounter ids.
USTAR tarformat
Unix Standard Tar (POSIX 1003.1). The lowest-common-denominator archive format. Asclepius evidence bundles are USTAR so they can be opened with the system tar from any UNIX of the past 30 years.

V

verify()function
The chain-walking integrity check. Ledger::verify() returns Ok iff every entry's seq is monotonic, every prev_hash matches the previous entry's entry_hash, every payload_hash is a correct re-hash of the body, and every signature verifies against the ledger's public key.
