conformance suite · v0.1.0 · 2026-04-25

Three levels. Each one runs.

The conformance suite is the third-party-runnable test battery that decides whether a vendor integration meets L1 audit, L2 drift, or L3 prospective evidence. Each level strictly extends the previous. Vendors advertise the highest level they pass; customers buy at the level their AI Governance Office requires. The full reference suite is tests/asclepius_tests in the repo — these are the externally-defined assertions that any compliant runtime must satisfy.

L1

audit

Trustworthy log.

10 / 10 pass

The runtime emits one entry per wrapped inference. The chain verifies. Bundles export and reverify offline. The signing key is properly held and never leaks into logs. L1 is the floor: without it, nothing else matters.

chain.append-onlyThe runtime exposes no API to delete or modify a committed entry. Attempting it via direct DB access is detectable by verify(). spec § ledger-entry
chain.monotonic-seqFor every entry at sequence N, the next entry has sequence N+1. Gaps are integrity failures. test_audit.cpp
chain.prev-hash-bindsEach entry's prev_hash equals entry_hash(N-1). Mutating any past entry changes its hash and breaks the chain at the next step. threat § A1
chain.signature-bindsEach entry carries an Ed25519 signature over (seq, ts, prev, payload, actor, type, tenant, body). Verifies against the registered public key. spec § signature
canonical-json.deterministicTwo equal-shaped bodies produce byte-equal JSON. Sorted keys, no whitespace, UTF-8. spec § canonical-json
bundle.signed-by-ledger-keyThe exported bundle's manifest.sig verifies against the ledger's public key, not a throwaway. bundle.cpp
bundle.contents-match-manifestEvery file declared in the manifest's files map has its hash present and verifiable. spec § bundle
bundle.offline-verifiableThe verifier requires only the bundle file and the public key; no network, no service, no vendor SaaS. verify ritual
phi.not-in-ledger-by-defaultInference body stores hashes of input/output, not content. Opt-in is the only way to put PHI into the chain. threat § A5
consent.scope-enforcedAn inference is rejected before any policy runs or any model is called if there's no active, non-expired consent token for the patient and purpose. test_runtime.cpp

L2

drift

Observable behaviour.

4 / 4 pass

L1 plus continuous drift telemetry, override capture, and a runtime that surfaces severity transitions instead of dashboards alone. The model can be retired without losing replayability.

drift.psi-correctPopulation Stability Index implementation matches the reference (sum over bins of (p-q)·ln(p/q)) within numerical noise. test_telemetry.cpp
drift.severity-bandsThresholds match the spec: PSI < 0.10 stable, < 0.25 minor, < 0.50 moderate, ≥ 0.50 severe. spec
override.captured-with-rationaleAn override emits an evaluation.override entry tying back to the original inference_id, including the corrected payload and a non-empty rationale. spec § events
rotate.preserves-baselineDriftMonitor::rotate() resets the current window without clearing the baseline. PSI is computed against the baseline always, so slow drift accumulates. drift.cpp

L3

evidence

Prospective trial.

3 / 5 partial

L2 plus ground-truth coupling on a defined fraction of inferences. Per-window comparative metrics. Bundle suitable for an FDA submission, signed and reproducible. The substrate becomes the trial harness.

ground-truth.capturedattach_ground_truth() writes an evaluation.ground_truth entry to the chain, tying back to the original inference. harness.cpp
metrics.window-boundedmetrics({start, end}) includes only entries whose ts ∈ [start, end). A bundle's metrics depend only on its window, not on the chain's full length. harness.cpp
bundle.includes-prospective-streamsAn exported bundle for a window with overrides + ground-truth contains both overrides.jsonl and ground_truth.jsonl, signed by the same key as the manifest. bundle.cpp
reproducibility.bit-equalRe-exporting a closed window must produce a byte-identical bundle (modulo the manifest signature, which is non-deterministic for Ed25519). Implementation in v0.2. planned
fda.precert-shapeThe bundle's manifest schema includes the fields an FDA Predetermined Change Control Plan submission references (model id, version, training date, prospective window). Schema land Q4 2026. planned
