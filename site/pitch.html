What is this thing, exactly?

Asclepius is a C++20 library you link against. It is not a hospital, not an EHR, not a model. It wraps any clinical-AI tool running in an Indian deployment and turns every inference into a chained, signed, regulator-readable event.

This page is the story, told as it happens. A clinic in Bengaluru. A model. A decision. The eight things the substrate does, scene by scene, before the patient leaves the room.

§01 — the setting

A clinic in Bengaluru.

A 72-year-old patient. Dr. Iyer wants a faster read on a chest X-ray. The hospital has a screening model — built by an ABDM-listed AI vendor, signed off by the radiology HOD, deployed last quarter. It runs in <200 ms.

The model decides. The patient nods. The line moves forward. Today, in Bengaluru, the kernel is not in the loop.

question What did the model see, what did it say, who was the patient, what consent was on file, and where is the trace?

§02 — the gap

The inference vanishes.

No record of consent. No fingerprint of the model version. No hash of the input. No hash of the output. Three months from now, when a DPDP officer asks "who saw which X-ray when, on what authority?" — the answer is a SQL row that says 2026-04-29 11:42 inference and an exhausted ops engineer.

This is the failure mode every Indian clinical-AI deployment will eventually meet — at NABH audit, at CDSCO SaMD review, or in court.

cost A retroactive evidence pipeline costs 6–12 engineer-months, and it can never reconstruct what wasn't logged at the moment of the call.

§03 — wrap

Asclepius wraps the model.

One line: Runtime::open(db).infer(ctx). Returns an Inference handle. Now the kernel is in the loop. Consent is checked against the patient's ABHA-linked token before the model runs. PHI is scrubbed from the input. The model still decides; it just decides inside a substrate.

The kernel does not look at the model's weights. It does not have an opinion about the diagnosis. It only ensures the call happens with consent, with PHI removed, and with a hash on the way in and on the way out.

integration Add -lasclepius to the link line. Existing model code unchanged.

§04 — chain

Every call chains.

A new ledger entry. seq = previous + 1. prev_hash = BLAKE2b-256 of the entry before. Body = canonical JSON: actor, patient, encounter, model id, input_hash, output_hash, latency. Ed25519 signature over the whole row. SQLite WAL append. ~50 µs per entry on a modern laptop.

One year of inference at 10/sec is roughly 300 million rows. The chain still verifies in one ledger.verify() call.

property Mutate any byte of any row and the chain breaks at that row's seq. Tampering is detectable, not mitigable.

§05 — verify

One call. Verifies.

ledger.verify(). Returns OK or fails with the seq at which the chain breaks. No off-system tooling, no third-party SaaS. The signing key sits in a 0600 sidecar in dev or a HSM-fronted KeyStore in production. Apache 2.0. Vendor-neutral. Built on libsodium; no rolled crypto.

L1 / L2 / L3 / L2-Medical conformance ladders. The current reference build verifies a 2,000-entry chain in ~103 ms end-to-end.

tagline The Linux kernel for clinical AI. The same framing applies everywhere; India is where we lead.

§06 — india profile

India-shaped primitives.

The same substrate, switched into the India profile: ABDM consent artefacts that round-trip through the national Health Information Provider gateway. FamilyGraph for adult-children consenting on behalf of elder parents — the eldercare workflow no Western kernel has. EmergencyOverride implementing DPDP § 7 break-glass with a 72-hour mandatory backfill.

Plus HumanAttestation, BillAuditBundle, TeleConsultEnvelope, SampleIntegrityBundle, CarePathAttestation — five signed evidence shapes covering the workflows from Razorpay's "Fix My Itch" health list.

opt-in A US or EU deployment never instantiates these types. The India profile compiles in but is opt-in per primitive.

§07 — bundle

One bundle. Regulator-portable.

When the auditor asks, asclepius bundle > case.tar emits a USTAR archive with the manifest, the ledger excerpt for the time window, the metrics snapshot, the drift report, the override and ground-truth files, the public key, and a manifest signature.

The auditor verifies offline. No network, no SaaS dependency, no per-seat licence. Evidence is the deliverable.

format Plain USTAR tar. Anyone with libsodium can verify the manifest signature. Spec is one page in SPEC.md.

§08 — the regulator

CDSCO. NABH. DPDP DPB.

The auditor runs asclepius evidence verify case.tar. Returns OK or a precise reason it does not. Same binary the operator built. Same crypto. The chain of custody is the product.

When CDSCO finalises its SaMD framework, when the DPDP Data Protection Board issues its first compliance order, when an NABH inspector wants to know what consent was on file at 11:42 ist on April 29 — the answer is a single tar and a single command.

candid risk The Indian regulatory demand we are building for is real but maturing. CDSCO SaMD is draft; DPDP Board is staffing up. The kernel works in any jurisdiction today; India is where we believe the demand lands first because the rails (ABDM) are already there.

One library. Eight scenes. Linkable today.

v0.6.0. Apache 2.0. 1,565 tests, 36,098 assertions, zero failures. Build it, link it, run the end-to-end India journey example (examples/08_india_full_journey). Eight ledger entries, one chain-verify, < 1 second wall-clock.
