§01 · metrics · traces · logs · alerts

What an operator should watch.

Asclepius emits Prometheus-shaped metrics, OpenTelemetry-conventioned traces, and JSON Lines logs. Nothing magic — everything stock. The four sections below enumerate every signal the substrate produces, with example PromQL alert rules that pair to the runbooks in playbooks.

Metrics — Prometheus shape.

Every metric is prefixed asclepius_. Counters end in _total; histograms in _seconds or _bytes; gauges have no suffix. Cardinality is bounded: label sets are limited to actor scheme (not actor id), policy id (not payload), and feature id (not feature value).

§ 02 · OpenTelemetry · OTLP/gRPC :4317

Traces — OpenTelemetry semantic conventions.

Span names follow OTel {component}.{operation}. The runtime propagates trace context across the policy chain and the model call so you can see exactly where the latency lives. The model call is its own span — vendor-agnostic, always there.

spanparentattributes
asclepius.inferencerootTop-level inference span. Attributes: asclepius.purpose, asclepius.actor.scheme, asclepius.model.id, asclepius.model.weights_sha256.
asclepius.consent.checkinferenceasclepius.consent.purpose, asclepius.consent.result.
asclepius.policy.runinferenceasclepius.policy.id, asclepius.policy.version, asclepius.policy.verdict.
asclepius.model.callinferenceThe model boundary. Operator instruments the body. Attributes: asclepius.model.id.
asclepius.drift.updateinferenceasclepius.drift.feature, asclepius.drift.psi.
asclepius.ledger.appendinferenceasclepius.ledger.seq, asclepius.ledger.kind.
asclepius.crypto.signledger.appendasclepius.crypto.alg = ed25519, asclepius.key.id.
asclepius.bundle.exportrootOut-of-band; root span when an export is initiated.
asclepius.verify.chainrootWatchdog or on-demand verify. asclepius.verify.scope, asclepius.verify.result.

What we don't trace. Anything containing PHI. The model call's request/response body is not an attribute — only its hash is. The same is true for the inputs of the policy chain. The trace is operational, the ledger is forensic.

§ 03 · stdout · JSON Lines · 1 line per event

Logs — JSON Lines, structured.

The substrate writes one JSON object per line to stdout. Operators wire that to whatever ingestor they prefer (Vector, Fluent Bit, journald). No free-text strings. Every log line has a code a runbook can look up. 

{ "ts": "2026-04-26T14:21:08.412Z",
  "level": "info",
  "code": "inference.completed",
  "actor": "clin:np_marisol_h",
  "purpose": "scribe",
  "model": { "id": "scribe-v3", "weights_sha256": "6a0f…e2" },
  "latency_us": 147,
  "ledger_seq": 874,
  "trace_id": "4f1e…",
  "span_id": "a92c…" }

Notable codes (each maps to a section of playbooks):

codelevelwhat it means
inference.completedinfoSuccessful end-to-end inference. Routine.
policy.violationwarnAn input or output failed a policy. Includes policy_id and verdict.
consent.deniedwarnInference rejected due to missing/revoked consent.
chain.brokenerrorP1. Ledger verify failed. Triggers PB-01.
drift.psi.breachwarnA feature's PSI crossed threshold. PB-04.
key.rotatedinfoScheduled rotation succeeded. PB-02.
key.revokederrorP1. Emergency revocation. PB-03.
backend.unreachablewarnSidecar can't reach central backend; running on local WAL. PB-07.
bundle.exportedinfoEvidence bundle written. Routine for compliance work.
§ 04 · pager-ready · paste into prometheus

Alert rules.

Twelve PromQL rules sufficient to run a single tenant. Severity tags align with the runbook severities in playbooks. Operators almost always want to layer their own org-wide rules on top of this baseline. 

# PB-01 — chain broken — never zero increment is acceptable
- alert: AsclepiusChainBroken
  expr:  increase(asclepius_ledger_chain_broken_total[5m]) > 0
  for:   0m
  labels: { severity: P1 }
  annotations: { runbook: "playbooks.html#truncation" }

# PB-03 — key rotation due / overdue (180-day window)
- alert: AsclepiusKeyRotationDue
  expr:  asclepius_key_age_seconds > 180 * 86400
  for:   1h
  labels: { severity: P3 }
  annotations: { runbook: "playbooks.html#key-rotation" }

# PB-04 — drift PSI breach for > 30 min in any feature
- alert: AsclepiusDriftBreach
  expr:  max by (feature) (asclepius_drift_psi) > 0.25
  for:   30m
  labels: { severity: P2 }
  annotations: { runbook: "playbooks.html#drift-breach" }

# PB-05 — consent revocation propagation SLA (60 s p99)
- alert: AsclepiusConsentLag
  expr:  histogram_quantile(0.99, sum by (le) (rate(asclepius_consent_revocation_propagation_seconds_bucket[10m]))) > 60
  for:   10m
  labels: { severity: P2 }
  annotations: { runbook: "playbooks.html#consent-revocation" }

# PB-07 — backend unreachable for > 60 s
- alert: AsclepiusBackendUnreachable
  expr:  absent_over_time(asclepius_ledger_seq[60s])
  for:   1m
  labels: { severity: P2 }
  annotations: { runbook: "playbooks.html#partition" }

# bundle self-verify failure — should be zero ever
- alert: AsclepiusBundleSelfVerifyFailed
  expr:  increase(asclepius_bundle_verify_failures_total[15m]) > 0
  for:   0m
  labels: { severity: P1 }

# inference latency p99 > 500ms — soft alert, investigate the chain
- alert: AsclepiusInferenceLatencyHigh
  expr:  histogram_quantile(0.99, sum by (le, purpose) (rate(asclepius_inference_latency_seconds_bucket[5m]))) > 0.5
  for:   10m
  labels: { severity: P3 }

# policy violation rate > 5% in 5 min — model drift OR adversarial input
- alert: AsclepiusPolicyViolationsElevated
  expr:  sum(rate(asclepius_policy_violations_total[5m])) / sum(rate(asclepius_inferences_total[5m])) > 0.05
  for:   10m
  labels: { severity: P2 }

# consent denied rate > 1% — process problem, not substrate problem
- alert: AsclepiusConsentDeniedElevated
  expr:  sum(rate(asclepius_consent_checks_total{result="denied"}[5m])) / sum(rate(asclepius_consent_checks_total[5m])) > 0.01
  for:   15m
  labels: { severity: P3 }

# ledger size growth alarm (4× expected weekly rate)
- alert: AsclepiusLedgerGrowthAnomaly
  expr:  deriv(asclepius_ledger_size_bytes[1d]) > 4 * deriv(asclepius_ledger_size_bytes[7d] offset 7d)
  for:   1h
  labels: { severity: P3 }

# sign latency regression — usually means HSM trouble
- alert: AsclepiusSignLatencyRegression
  expr:  histogram_quantile(0.99, sum by (le) (rate(asclepius_sign_seconds_bucket[5m]))) > 0.005
  for:   10m
  labels: { severity: P3 }

# runtime read-only flag should be zero in steady state
- alert: AsclepiusReadOnlyAsserted
  expr:  asclepius_runtime_read_only == 1
  for:   5m
  labels: { severity: P2 }

Pair with playbooks. Every runbook above pages out to a specific runbook URL via the annotations.runbook field. Your paging tool (PagerDuty, Opsgenie, FireHydrant) can link straight to "what to do at 3 a.m."

Visibility is the other half of trust.

A substrate that emits cryptographic provenance but no operational signal is a black box that happens to be auditable in retrospect. That is not enough. The runbook companion to "what does the chain prove?" is "what does the system look like, in real time, while it is proving it?"

This page is the answer. Wire it into your stack of choice; the runbooks take it from there.
