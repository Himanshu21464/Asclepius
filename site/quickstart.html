§01 · five minutes · pick your stack

From zero to a signed inference.

Install Asclepius, open a runtime, push a policy chain, wrap one inference, verify the chain. Five minutes, four steps, four stacks. By the end you have a SQLite ledger with one signed entry and a verify command that prints ✓ chain ok · 1 entries. That is the substrate working.

1minstall
1mopen + chain
2mwrap inference
1mverify

Pick your stack.

step 1 · install

Get the library.

Asclepius ships as a CMake-friendly C++20 library with libsodium and SQLite as the only required dependencies.

# Linux (Ubuntu 24.04 +):
sudo apt install libasclepius-dev
# or via FetchContent in your project's CMakeLists:
FetchContent_Declare(asclepius
    GIT_REPOSITORY https://github.com/Himanshu21464/Asclepius.git
    GIT_TAG        v0.1.0)
FetchContent_MakeAvailable(asclepius)
target_link_libraries(myapp PRIVATE asclepius::asclepius)
step 2 · open + chain

Open a runtime, push policies.

The runtime owns one ledger. Policies are pushed in execution order.

#include <asclepius/asclepius.hpp>

asc::Runtime rt = asc::Runtime::open("./asclepius.db").value();
rt.policies().push(asc::policy::PhiScrub::v4());
rt.policies().push(asc::policy::SchemaTranscript::v2());
rt.policies().push(asc::policy::Length::v1(8192));
step 3 · wrap an inference

Sign one entry.

The model lives inside the inference scope. Asclepius scrubs the input, runs your model, signs the output.

auto inf = rt.begin_inference({.actor="clin:dr_marisol",
                                .patient=PatientId("pt:demo"),
                                .model={"scribe-v3", "6a0f…e2"},
                                .purpose=asc::Purpose::Scribe});
auto safe = inf.policy_chain(transcript);
auto note = my_scribe::transcribe(safe);
auto [seq, sig] = inf.commit(note);

std::cout << "committed seq=" << seq << "\n";
step 4 · verify

Confirm the chain.

auto r = rt.ledger().verify();
if (r.ok()) std::cout << "✓ chain ok · " << r.value().count << " entries\n";
else        std::cerr << "✗ chain broken: " << r.error().message << "\n";
✓ chain ok · 1 entries

Context-manager scope.

with rt.begin_inference(actor="clin:dr_marisol",

Five things that will trip you up.

The substrate is small but opinionated. These are the patterns that reliably waste the first hour. Read them before you debug — most problems are one of these.

policy chain ordering

Policies run in the order pushed. PhiScrub must come before any policy that hashes input — otherwise the hash binds to PHI bytes that should never have been hashed.

actor scheme prefix

Actor ids must match (clin|svc|sys|agent):name. Plain strings are rejected. See audit-entry schema.

weights_sha256 changes

If your model's weights file changes byte-for-byte, the hash changes and that is recorded — see TT-03. Always pin and verify in production.

SQLite single writer

One writer per ledger. Two processes opening the same DB will deadlock on append. The runtime is a long-lived handle; share it via your service framework.

commit() is non-throwing

Returns Result<Committed, Error>. Check .ok(). Letting the result destruct without inspection is a build warning; don't ignore it.

HSM optional, recommended

The default keystore is on-disk for development. For production, point KeyStore at an HSM-backed implementation — see deploy · PB-02.
